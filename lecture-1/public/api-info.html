API Endpoints

Method Endpoint Description
GET /users Get all users
POST /users Register a new user
GET /users/:id/details Get user with company details
GET /companies/:id/users Get company with its users
GET /profiles Get all profiles
POST /profiles Create a profile
POST /profiles/seed Seed profiles (for testing)
POST /auth/login-bcrypt Login with bcrypt (no JWT)
POST /auth/login-jwt Login and get JWT token
POST /auth/login-session Login and create session

Detailed API Documentation

GET /users

200 OK
Request: None
Response:
{
  "success": true,
  "message": "All user data Retrived sucessfully ...",
  "data": [ /* Array of users */ ]
}

POST /users

201 Created
Request:
{
  "userName": "John Doe",
  "userEmail": "john@example.com",
  "userId": 1
}
Response:
{
  "success": true,
  "message": "User Created sucessfully ...",
  "data": {
    "userId": 1,
    "userName": "John Doe",
    "userEmail": "john@example.com"
  }
}

GET /users/:id/details

200 OK | 404 Not Found
Request: None
Response:
{
  "success": true,
  "data": { /* User object with companyDetails */ }
}

POST /auth/login-bcrypt

200 OK | 401 Unauthorized
Request:
{
  "email": "john@example.com",
  "password": "yourpassword"
}
Response:
{
  "success": true,
  "message": "Login successful",
  "data": {
    "id": 1,
    "email": "john@example.com"
  }
}

POST /auth/login-jwt

200 OK | 401 Unauthorized
Request:
{
  "email": "john@example.com",
  "password": "yourpassword"
}
Response:
{
  "success": true,
  "message": "Login successful",
  "data": {
    "token": "JWT_TOKEN"
  }
}

POST /auth/login-session

200 OK | 401 Unauthorized
Request:
{
  "email": "john@example.com",
  "password": "yourpassword"
}
Response:
{
  "success": true,
  "message": "Login successful (session)",
  "data": {
    "id": 1,
    "email": "john@example.com"
  }
}

Usage

  • All endpoints accept and return JSON.
  • Authentication endpoints support bcrypt, JWT, and session-based login.
  • See /profiles/seed for test data seeding.
  • Use Authorization: Bearer <token> for JWT-protected endpoints (if implemented).
